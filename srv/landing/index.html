The Problem

today means stitching
5 different services

  • ✕ OpenAI API for chat, Anthropic for Claude, Groq for speed — three different SDKs, three billing dashboards.
  • ✕ ChromaDB / Pinecone / Weaviate for vector search — yet another service to provision, monitor, and pay for.
  • ✕ Redis / PostgreSQL for conversation memory — schema design, connection pooling, backup scripts.
  • ✕ Custom MCP server for tool calling — JSON-RPC boilerplate, auth, error handling from scratch.
The Fix

The Solution

φ Gateway — all four primitives, one API.

  • ✓ One endpoint for GPT-5.5, Claude Opus 4.6, Llama 4, Gemini 2.5, Mistral Large — auto-routed by model name. Bring your own keys.

Four Primitives

Everything your agent needs.
Nothing it doesn't.

No LangChain. No ChromaDB. No Redis. No separate MCP server. Just one API — built to run on a 4GB VPS.

LLM Proxy

Route to 16+ models across OpenAI, Anthropic, Groq, and OpenRouter through one OpenAI-compatible endpoint. Use any OpenAI SDK — just change the base URL. Every response includes cost_usd so your agent tracks spending.

GPT-5.5Claude Opus 4.6Llama 4Gemini 2.5Mistral Large+11 more
POST /v1/chat/completions streaming (SSE) supported

Tool Registry + MCP

Register tools with JSON Schema contracts. Agents auto-discover them via the MCP endpoint (JSON-RPC 2.0). No custom MCP server to build — just POST your tool definition and it's live. Supports none, api_key, and oauth auth types.

tools/listtools/callresources/read
POST /v1/tools · POST /mcp

Knowledge Base (RAG)

Ingest documents → auto-chunk by paragraph → embed via OpenAI → semantic cosine similarity search. No separate vector database — pure Python, zero native dependencies. Falls back to keyword search when no embedding key is configured. Each KB is scoped to an API key.

auto-chunkembedsemantic searchkeyword fallback
POST /v1/kb · POST /v1/kb/{id}/search

Agent Memory

Persistent conversation storage with automatic context window management. Tracks token counts per message. When a conversation exceeds the model's context limit, oldest messages are trimmed automatically. The X-Context-Truncated header warns your agent before data loss.

session persistenceauto-trimtoken tracking
POST /v1/memory/conversations · GET /v1/memory/conversations/{id}/messages
