## WHAT THIS PROJECT IS (AND IS NOT)

### IS:
```yaml
phi_gateway_is:
  - "API platform that provides primitives for AI agent developers"
  - "LLM proxy to multiple providers with unified interface"
  - "Tool registry where agents discover and call tools (MCP-native)"
  - "Knowledge base API for RAG (document ingest + semantic search)"
  - "Agent memory API (session + persistent cross-session)"
  - "Self-hosted, lightweight, runs on small VPS"
  - "Priced in IDR for Indonesian developers"
  - "Open source core with optional managed hosting"
```

### IS NOT:
```yaml
phi_gateway_is_not:
  - "An AI agent itself (it's infrastructure FOR agents)"
  - "A chatbot or personal assistant (that's OpenClaw/Hermes)"
  - "A no-code platform (target users are developers who write code)"
  - "An LLM training/fine-tuning service"
  - "A LangChain/LangGraph competitor (it's infrastructure layer, not framework)"
  - "A consumer product (B2B developer tools)"
  - "OpenAI/Anthropic wrapper with markup (value-add is tool registry + KB + memory)"
```

## KEY DECISIONS (TL;DR)

| # | Decision | Why | See |
|---|----------|-----|-----|
| 1 | **Path A — pure infrastructure** | Build own platform, not depend on OpenClaw/Hermes | ADR-001 |
| 2 | **Python + FastAPI** over Go/Rust | Speed of development, AI ecosystem, async native | ADR-002 |
| 3 | **SQLite + pure Python vec** over PostgreSQL + ChromaDB | Zero ops, no native deps, 200MB saved vs sqlite-vec | ADR-003 |
| 4 | **Caddy** over Nginx | Auto SSL, simpler config, single binary | ADR-004 |
| 5 | **Proxy-first** (not local LLM) | 4GB constraint, better to proxy Groq/OpenAI | ADR-005 |
| 6 | **MCP protocol** from day 1 | Becoming de facto standard for agent-tool communication | ADR-006 |
| 7 | **Beachhead: Indo AI devs** | Underserved, no local competitor, IDR pricing advantage | ADR-007 |
